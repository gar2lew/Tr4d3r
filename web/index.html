    No live order attempts yet this session.
    About blank TP / SL on OKX: this build uses bot-managed protective exits. The buy order is sent to OKX as a plain market buy, and the bot then watches every price tick and submits the sell when SL / TP / trail is hit. That means OKX order history will show TP and SL as blank on the original buy — this is expected, not a bug. The protective exits will appear in OKX history only when the bot actually fires them. If the bot stops while a live position is open, your stop is no longer being watched. V9: Native exchange-attached TP / SL (algo orders) is implemented and runs alongside bot-managed exits when LIVE_NATIVE_PROTECTION_ENABLED=true. Even with native on, you should still keep this app running for trailing stops and event logging.
    V9.1 hotfix: live sells now use a sellable quantity that subtracts any base-asset fee OKX deducted on the buy fill, and the sell qty is clamped to the smaller of journal sellable vs OKX free balance, rounded down to lotSz. If the clamped qty falls below minSz, the bot refuses the sell and latches the kill switch with a clear reason instead of looping on rejections. Existing positions are clamped to OKX inventory on refresh/startup; the kill switch is never auto-released.
